# LingoAI ASR→LLM→TTS 管道实现文档

**版本**: v2.0  
**更新日期**: 2026-06-07  

---

## 1. 概述

ASR→LLM→TTS 管道（Pipeline）是 LingoAI 的核心 AI 对话编排系统，实现"语音识别 → 大语言模型推理 → 语音合成"的全链路串行处理。用户可通过前端设置自由启用/禁用管道的各阶段，灵活组合不同的 AI 引擎。

### 1.1 设计目标

- **模块化**: ASR、LLM、TTS 三个阶段可独立替换引擎
- **可配置**: 每个会话可独立设置管线参数（引擎、模型、角色等）
- **可观测**: 提供完整的延迟指标和运行状态监控
- **流式响应**: 支持 Server-Sent Events (SSE) 流式输出

### 1.2 管道流程

```
     [用户输入]
         │
         ▼
    ┌─────────────┐
    │  ASR 阶段   │ ← 可选：豆包 / Whisper / Azure / Faster-Whisper
    │ (语音→文本) │
    └──────┬──────┘
           │ (文本)
           ▼
    ┌─────────────┐
    │  LLM 阶段   │ ← 可选：OpenAI / DeepSeek / 智谱GLM / 通义千问 / 豆包
    │ (文本→回复) │
    └──────┬──────┘
           │ (回复文本 + 翻译 + 语法分析)
           ▼
    ┌─────────────┐
    │  TTS 阶段   │ ← 可选：OpenAI TTS / Piper / Edge TTS
    │ (文本→音频) │
    └──────┬──────┘
           │ (音频 URL 或流)
           ▼
     [前端输出: 文本 + 音频]
```

## 2. 核心组件

### 2.1 ConversationPipelineService（管线编排器）

**类路径**: `com.lingoai.service.ai.ConversationPipelineService`

这是管线的核心编排器，负责按顺序调用 ASR → LLM → TTS 三个阶段，并组装最终响应。

#### PipelineConfig（管线配置）

```java
@Builder @Data
public static class PipelineConfig {
    private boolean useAsr;        // 是否启用 ASR 阶段
    private boolean useTts;        // 是否启用 TTS 阶段
    private String agentName;      // AI 角色名（Ava / Andrew / Emma / Interviewer）
    private String asrEngine;      // ASR 引擎: doubao | azure | whisper
    private String llmEngine;      // LLM 引擎: openai | deepseek | glm | qianwen | doubao
    private String llmModel;       // LLM 模型名称
    private String llmApiKey;      // LLM API Key（可选）
    private String llmBaseUrl;     // LLM Base URL（可选）
    private Double llmTemperature; // LLM 温度参数
    private String ttsEngine;      // TTS 引擎: openai | enjoyai | piper | edge-tts
    private String ttsModel;       // TTS 模型
    private String ttsVoice;       // TTS 音色
}
```

#### PipelineResult（执行结果）

```java
@Builder @Data
public static class PipelineResult {
    private String userText;           // ASR 处理后的用户文本
    private String aiResponseText;     // LLM 生成的 AI 回复
    private String translatedText;     // 中文翻译（可选）
    private String analysisText;       // 语法分析文本（可选）
    private String ttsUrl;             // TTS 音频 URL（可选）
    private String agentName;          // 当前 AI 角色名
    private String agentDescription;   // 角色描述
}
```

### 2.2 LlmService（大语言模型服务）

**类路径**: `com.lingoai.service.ai.LlmService`

负责与多个 LLM 引擎的通信，支持：
- **同步调用** (`chat`)：完整生成后返回
- **流式调用** (`streamChat`)：通过 Consumer 逐 token 回调
- **多引擎适配**：自动根据 `engine` 参数选择对应的 API 客户端

#### 支持的 LLM 引擎

| 引擎标识 | 服务商 | 默认模型 |
|---------|--------|---------|
| `openai` | OpenAI | gpt-4o |
| `deepseek` | 深度求索 | deepseek-v4-flash |
| `glm` | 智谱 AI | glm-5.1 |
| `qianwen` | 阿里云通义千问 | qwen-2-plus |
| `doubao` | 字节跳动豆包 | doubao-pro |
| `ollama` | 本地 Ollama | llama3 |

#### 流式调用接口

```java
public void streamChat(
    List<Map<String, String>> messages,  // 对话历史
    String systemPrompt,                 // 系统提示词
    String model,                        // 模型名称
    double temperature,                  // 温度参数
    String apiKey,                       // API Key（可选覆盖）
    String baseUrl,                      // Base URL（可选覆盖）
    String engine,                       // 引擎标识
    Consumer<String> onMessage           // 流式回调
)
```

### 2.3 TtsService（语音合成服务）

**类路径**: `com.lingoai.service.ai.TtsService`

#### 支持的 TTS 引擎

| 引擎标识 | 类型 | 说明 |
|---------|------|------|
| `openai` | 云端 | OpenAI TTS API |
| `enjoyai` | 云端 | OpenAI 兼容 API |
| `piper` | 本地离线 | Piper 神经网络 TTS（通过 LocalTtsClient 调用） |
| `edge-tts` | 本地在线 | Microsoft Edge TTS（通过 LocalTtsClient 调用） |

### 2.4 AsrService（语音识别服务）

**类路径**: `com.lingoai.service.ai.AsrService`

支持豆包实时语音识别、Azure 语音识别和本地 Faster-Whisper。

### 2.5 SessionCoordinator（会话协调器）

**类路径**: `com.lingoai.service.ai.SessionCoordinator`

管理每个对话会话的状态机，协调 ASR、LLM、TTS 的流式交互。

#### 状态机

```
LISTENING → THINKING → SPEAKING → LISTENING (循环)
    ↑                        │
    └────────────────────────┘ (打断: SPEAKING → LISTENING)
```

| 状态 | 说明 |
|------|------|
| `LISTENING` | 监听用户语音输入 |
| `THINKING` | 调用 LLM 生成回复 |
| `SPEAKING` | 播放 TTS 语音回复 |

#### PipelineMetrics（延迟指标）

```java
public class PipelineMetrics {
    private long llmFirstTokenLatency;  // LLM 首 token 延迟 (ms)
    private long ttsFirstAudioLatency;  // TTS 首音频延迟 (ms)
    private long endToEndLatency;       // 端到端延迟 (ms)
    private long asrLatency;            // ASR 识别延迟 (ms)
}
```

## 3. Agent 角色系统

**类路径**: `com.lingoai.constants.AgentTemplates`

系统内置预定义的 AI 角色模板，每个角色包含系统提示词、默认参数和 TTS 音色配置。

### 3.1 内置角色

| 角色名 | 语言 | 风格 | TTS 音色 |
|--------|------|------|----------|
| Ava | 英语 | 友好、鼓励型英语导师 | en_US-amy-medium |
| Andrew | 英语 | 专业、严谨型英语导师 | en_US-lessac-medium |
| Emma | 英语 | 温柔、耐心型导师 | en_US-libritts-high |
| Interviewer | 英语 | 正式、中立型面试官 | en_US-amy-medium |

### 3.2 角色模板格式

每个角色定义如下：

```java
public static final AgentInfo AVA = AgentInfo.builder()
    .name("Ava")
    .description("A friendly and encouraging English tutor")
    .language("en")
    .systemPrompt("You are Ava, a friendly English tutor...")
    .temperature(0.8)
    .ttsVoice("en_US-amy-medium")
    .ttsModel("tts-1")
    .build();
```

## 4. AI 管线设置管理

**类路径**: `com.lingoai.service.ai.AiPipelineSettings`

管线设置存储在 Redis 中，支持两级配置：

| 级别 | 键模式 | 生命周期 |
|------|--------|----------|
| 用户级 | `pipeline:config:user:{userId}` | 持久化 |
| 会话级 | `pipeline:config:session:{sessionId}` | 会话期间 |

配置加载优先级：会话级 > 用户级 > 默认配置

## 6. 管道调用流程（完整链路）

### 6.1 正常对话流程

```
1. 前端 POST /api/practice/chat
       │
2. ConversationPipelineService.executePipeline()
       │
       ├── 3. [ASR 阶段] (如果启用)
       │     └── AsrService.transcribe(audioData) → userText
       │
       ├── 4. [LLM 阶段]
       │     ├── 加载 Agent 系统提示词
       │     ├── LlmService.chat(messages, systemPrompt, ...)
       │     └── → aiResponseText + translatedText + analysisText
       │
       ├── 5. [TTS 阶段] (如果启用)
       │     ├── TtsService.synthesize(aiResponseText, ...)
       │     └── → ttsUrl (音频文件路径)
       │
       └── 6. 组装 PipelineResult 返回
```

### 6.2 流式对话流程

```
1. 前端 GET /api/practice/chat/stream?text=... (SSE)
       │
2. PracticeController 创建 SseEmitter
       │
3. 异步线程执行:
       ├── 构建 messages (含 system prompt)
       ├── LlmService.streamChat(..., onMessage)
       │     └── onMessage 回调逐条发送 SSE event
       │           ├── event: message / data: {...token...}
       │           └── event: message / data: [DONE]
       └── emitter.complete()
```

### 6.3 打断流程（SessionCoordinator）

```
用户语音 → 前端 VAD 检测到说话
       │
前端发送 interrupt 控制帧
       │
后端 SessionCoordinator:
  ├── 停止 TTS 音频流
  ├── 取消 LLM 请求（关闭连接）
  ├── 清空缓冲区
  └── 状态切换: SPEAKING → LISTENING
       │
前端开始新语音采集
       │
新一轮对话开始
```

## 7. 配置管理

### 7.1 application.yml 管线配置

```yaml
ai:
  llm:
    default-engine: openai
    openai:
      api-key: ${OPENAI_API_KEY}
      base-url: https://api.openai.com/v1
      default-model: gpt-4o
      temperature: 0.8
    deepseek:
      api-key: ${DEEPSEEK_API_KEY}
      base-url: https://api.deepseek.com
      default-model: deepseek-v4-flash
    glm:
      api-key: ${GLM_API_KEY}
      base-url: https://open.bigmodel.cn/api/paas/v4
      default-model: glm-5.1
    qianwen:
      api-key: ${QIANWEN_API_KEY}
      base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
      default-model: qwen-2-plus

  tts:
    default-engine: piper
    local:
      base-url: http://localhost:8001
      piper-voice: en_US-amy-medium
      piper-speed: 1.0
      edge-voice: en-US-AriaNeural

  asr:
    default-engine: faster-whisper
    faster-whisper:
      host: localhost
      port: 50051
      model: large-v2
```

### 7.2 前端 AppConfig 管线配置

```typescript
interface PipelineConfig {
  enableAiPipeline: boolean;     // 总开关

  pipelineAgentName: string;     // AI 角色名

  pipelineTtsVoice: string;      // TTS 音色

}
```

## 8. 与端到端方案的对比

| 特性 | 管道式架构 (Pipeline) | 纯端到端模型（豆包 Realtime） |
|------|----------------------|------------------------------|
| 延迟 | ~1.2-2s（优化后） | <1s |
| 打断支持 | 通过 SessionCoordinator | 原生支持 |
| 模块可替换性 | 极高（各引擎独立） | 低（锁死厂商） |
| 发音评测集成 | 独立调用，不影响对话 | 需额外调用 |
| 成本控制 | 按需选择模型 | 单一模型定价 |
| 发音纠错 | ✅ 可深度集成 | ❌ 不支持 |
| 离线能力 | ✅ Piper TTS + Faster-Whisper 可离线 | ❌ 需联网 |
